Strategy patch

Velocity trigger for slow-burn 15-min windows

A second entry path that fires on rate of change, not absolute Delta — so windows that move 0.30% over 12 minutes still get traded, even when no single scan trips the static threshold.

Problem

The 07:30–07:45 window on 26 May 2026 dropped ~$252 (-0.33%) from open to close, but the bot stayed flat. Each minute's Delta was small in isolation; by the time Delta reached the SCALP entry threshold (~0.205%), the window was already in Phase4(closing) — past the entry cutoff. The move was real but its shape was wrong for the current logic.

A trade-worthy move on any reasonable strategy — directional, monotone for ~10 of 15 minutes.

Delta peaked at −0.249% in Phase2 — below the 0.205% threshold band only fractionally, and the larger −0.288% / −0.294% readings landed after Phase4 lockout.

The signal that was hiding in plain sight

Reading the per-minute scans laterally, the velocity of Delta — its first derivative over a rolling window — was clearly elevated well before T-200s:

From T-444s to T-262s — a span of ~3 minutes inside Phase2 — Delta moved from −0.086% to −0.235%, a swing of 0.149%. That's the same magnitude as the static threshold (0.205%), achieved in a third of the time, and it landed with 4+ minutes of trading runway left. The trade was there. The detector wasn't.

Specification

Trigger definition

Augment strategy_v2.py with a second SCALP entry path, evaluated in parallel with the existing absolute-Delta path. The trigger fires when the rolling velocity of Delta crosses a threshold and direction is consistent.

velocity_window_s
180seconds
Rolling lookback for velocity calc. ~3 scans at 60s cadence — long enough to filter noise, short enough to catch a 3-minute trend leg.
velocity_threshold
0.12% / window
|Δ_now − Δ_180s_ago| must exceed this. Calibrated so the 07:30 case triggers at T-262s with v₂ = 0.149%.
monotone_min
3scans
Of the last 4 scans, at least 3 must move in the same direction as the velocity. Rejects v-shaped chop that hits threshold via mean-revert.
entry_cutoff_s
T−240min
Velocity entries only valid until 4 min before close. Keeps a margin above Phase4 lockout for fill + execution risk.

Implementation

# ── add to StrategyState ──────────────────────────────────────────────
from collections import deque

class StrategyState:
    def __init__(self, ...):
        ...
        self.delta_history = deque(maxlen=8)  # (t_minus_s, delta_pct)

    def record_scan(self, t_minus_s: int, delta_pct: float):
        self.delta_history.append((t_minus_s, delta_pct))


# ── new trigger ───────────────────────────────────────────────────────
VELOCITY_WINDOW_S    = 180
VELOCITY_THRESHOLD   = 0.0012   # 0.12% over the window
MONOTONE_MIN         = 3        # of last 4 scans
VELOCITY_CUTOFF_S    = 240      # T-240, do not enter past this

def velocity_signal(state: StrategyState, t_minus_s: int) -> Optional[Direction]:
    """Return UP/DOWN if velocity trigger fires, else None.

    Fires when:
      1. |Δ_now − Δ_at_(t+window)| ≥ VELOCITY_THRESHOLD
      2. ≥ MONOTONE_MIN of last 4 scans agree in direction
      3. t_minus_s ≥ VELOCITY_CUTOFF_S (still safe to enter)
    """
    if t_minus_s < VELOCITY_CUTOFF_S:
        return None
    if len(state.delta_history) < 4:
        return None

    # find the scan closest to (t_minus_s + window) in the past
    target = t_minus_s + VELOCITY_WINDOW_S
    past = min(state.delta_history, key=lambda x: abs(x[0] - target))
    delta_now  = state.delta_history[-1][1]
    delta_past = past[1]
    velocity   = delta_now - delta_past

    if abs(velocity) < VELOCITY_THRESHOLD:
        return None

    # monotonicity check on last 4 scans
    last4 = list(state.delta_history)[-4:]
    diffs = [last4[i+1][1] - last4[i][1] for i in range(3)]
    direction = -1 if velocity < 0 else 1
    agree = sum(1 for d in diffs if (d >= 0) == (direction > 0))
    if agree < MONOTONE_MIN:
        log.info(f"VEL | rejected non-monotone — agree={agree}/3 v={velocity:+.4f}")
        return None

    log.info(
        f"VEL | trigger {Direction.DOWN if direction<0 else Direction.UP} "
        f"| v={velocity:+.4f} over {VELOCITY_WINDOW_S}s "
        f"| Δnow={delta_now:+.4f} Δpast={delta_past:+.4f} "
        f"| T-{t_minus_s}s"
    )
    return Direction.DOWN if direction < 0 else Direction.UP


# ── wire into the SCAN loop in Phase2 ─────────────────────────────────
def on_scan(self, t_minus_s: int, delta_pct: float):
    self.state.record_scan(t_minus_s, delta_pct)

    # Path A: existing absolute-Delta trigger
    direction = absolute_delta_signal(self.state, delta_pct)

    # Path B: new velocity trigger (only if A did not fire)
    if direction is None:
        direction = velocity_signal(self.state, t_minus_s)
        if direction is not None:
            self.state.last_trigger = "velocity"  # for P(win) selection

    if direction is not None:
        self.evaluate_entry(direction, t_minus_s)

Tradeoffs

What this buys

  • Catches slow-burn directional windows where the open-anchored Delta lags the actual trend
  • Preserves the existing absolute-Delta path entirely — purely additive, both can run in parallel
  • Monotonicity check rejects whippy chop that crosses threshold by accident
  • Cutoff at T−240s keeps execution runway above the Phase4 lockout, no rule conflict
  • Cheap to compute: 8-element deque, no extra HTTP, no new state across windows

What it costs

  • P(win) calibration table is fit against absolute-Delta cells — velocity entries will sit at a different point in (Delta, T-) space and may need their own calibration
  • Threshold 0.12% / 180s is back-fit to one window; needs validation across the full 90-day kline history before promotion to live
  • Late entries (T−240 to T−262) have less time to recover from an adverse maker fill — keep the $0.80 hard price cap firmly in place for velocity entries
  • Doubles the entry surface, so book_anomaly skips and thin-liquidity rejections will increase proportionally — monitor reject ratio after rollout

Rollout

  1. Backtest. Replay velocity_signal against the 90-day kline cache. Tag each fire with the realized 15-min outcome. Target: ≥55% directional accuracy at v=0.12%, monotone≥3/4.
  2. Shadow mode. Run the trigger in production but route fires to a log-only path (no orders). Compare velocity-fire windows vs absolute-Delta fires for 7 days. Look for overlap and disjoint coverage.
  3. Calibrate P(win). Build a second cell in the empirical table keyed by (T−, last_trigger=velocity). Use shrinkage toward the absolute-Delta prior while velocity sample size is small.
  4. Live, throttled. Enable for live trading with a per-window bet cap of half the normal SCALP size for the first 50 fires. Promote to full size only after P(win) confidence interval is tight.
  5. Monitor. Add log markers: VEL | trigger, VEL | rejected non-monotone, VEL | cutoff — so the next log review can audit reject reasons the way book_anomaly is audited today.
